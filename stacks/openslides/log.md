## Notes
- OpenSlides 4 uses a microservices architecture with 16 services
- The compose file is normally generated by the `openslides` manage tool via `openslides setup .`
- This compose.yaml was rendered from the Go template + default-config.yml defaults
- Images from ghcr.io/openslides/openslides (proxy, client, backend, autoupdate, search, projector, auth, vote, media, icc, manage)
- Uses Docker secrets for sensitive values (auth keys, passwords, certificates)
- Local HTTPS enabled by default with self-signed certificates
- Proxy listens on 127.0.0.1:8000
- Placeholder secret files created in ./secrets/ directory - must be replaced with real values for production
